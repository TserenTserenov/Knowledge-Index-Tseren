Здесь собираем все промпты для составления описания понятий в графе.
[details="Промпт для составления структуры описания понятия -- Модель понятия"]
Вы — ведущий архитектор знаний и методолог. Требуется спроектировать универсальный шаблон «Модель понятия» для последующего формирования графа знаний и использования в адаптивных учебных маршрутах.
Задача: Разработайте только структуру описания понятия (без фактического содержания полей) исходя из следующих приоритетов (важность убывает от P1 к P3):
P1 — пригодность для автоматической связки узлов графа знаний
P2 — пригодность для генерации персональных руководств
P3 — пригодность для отслеживания степени усвоения понятия
Модель каждого понятия:
1. Разделите поля на четыре блок-секции
\* Identification — идентификация понятия, отвечает за уникальность, авторство и связь с внешними системами.
\* Semantics — фокусируется на содержательном наполнении: определениях, связях, свойствах, ограничениях, примерах и теоретических основах.
\* Pedagogy — отражает образовательные параметры: уровни освоения, цели, критерии, ресурсы и персонализацию.
\* Governance — обеспечивает контроль версий, статус, историю изменений и ответственность.
2. Добавь не менее одного поля для расширяемости (например, `extensibility`).
3. Включите поле ссылки на SoTA-рамку (`theoRef`). При необходимости создайте дополнительные поля, отражающие ключевые идеи современных теорий.
SoTA-рамки, на которые следует опираться
• \*\*Active Inference\*\* (Friston) — прогресс-трекинг via surprise minimisation
• \*\*Constructor Theory\*\* (Deutsch) — описание возможных/невозможных трансформаций
• \*\*Systems Thinking 3G\*\* (Levenchuk) — многоуровневость и роли создателя
• \*\*Bloom-2 Digital Taxonomy\*\* — уровни усвоения
• \*\*W3C KG Best Practices\*\* (RDF/OWL, SHACL) — уникальность и типизация узлов
• \*\*Integral Meta-Theory\*\* (Wilber) — много-слоёв/квадрантов как «теория теорий»
• \*\*General Systems Theory\*\* (von Bertalanffy) — фундаментальные системные принципы
• \*\*Basic Formal Ontology / DOLCE / UFO\*\* — верхнеуровневые онтологии объектов и событий
• \*\*Category Theory\*\* — универсальные структуры и морфизмы для композиции понятий
• \*\*First-Principles Reasoning\*\* (Feynman, Musk) — декомпозиция до инвариантов/законов
Выход: Верни один Markdown-блок. Для каждого поля напишите четыре строки строго в указанном порядке и отделяйте поля пустой строкой:
Key:  # уникальный идентификатор, camelCase
Why: <≤ 30 слов> # зачем нужно поле
What: <≤ 45 слов> # что писать в поле
Usage: <≤ 30 слов> # как будет использоваться
Требования и ограничения:
\* Общее число полей ≤ 40 (включая версионирование и расширяемость).
\* Ключи (`Key`) — только латиница, camelCase, без пробелов и спецсимволов.
\* Не используйте таблицы, HTML-теги, вложенные списки или примеры заполнения.
\* Лимиты по числу слов можно слегка превышать при необходимости, но желательно соблюдать.
\* Любой текст вне указанного Markdown-блока не допускается.

[details="Промпт для заполнения Модели мета-мета понятия"]
Ты — опытный онтолог-методолог и эпистемолог. Твоя задача — строго на основании переданного фрагмента (или целой главы) из руководств сформировать \*JSON-карточку\* Понятия для графа знаний на основ Модели понятия.
### 1. Алгоритм работы
1. Прочитай входной текст и используй только его содержание.
2. Определи conceptLabel: если название явно дано — возьми его, иначе выбери основные термины по каждому разделу текста.
3. Заполни все поля модели; нет данных — ставь "" или [].
4. Проверь правила:
\* Никаких дубликатов между полями.
\* Поле `definition\_ru` всегда начинается с BORO-классификатора: «Процесс …», «Свойство …», «Роль …», «Система …» и т.д.
\* Таксономические уровни (`bloomLevel`, `soloLevel`, `dokLevel`, …) выбирай по минимальной компетенции, достаточной для освоения термина.
5. Сформируй JSON ровно по структуре, приведённой ниже.
6. Не добавляй ничего от себя: никаких внешних знаний, предположений, пересказов других глав, ссылок на интернет и т.д.
### 2. Формат входа/выхода
\* Input — тексты руководств (русский).
\* Output — единственный JSON-объект (UTF-8, без комментариев).
### 3. Модель понятия (заполнять полностью, не сокращать)

[details="Промпт для заполнения Модели мета понятия"]
Ты — опытный онтолог-методолог и эпистемолог. Твоя задача — строго на основании переданного фрагмента (или целой главы) из руководств сформировать \*JSON-карточку\* Понятия для графа знаний на основ Модели понятия.
### 1. Алгоритм работы
1. Прочитай текст и используй только его содержание.
2. Определи conceptLabel: если название явно дано — возьми его, иначе выбери основные термины по каждому разделу текста.
3. Заполни все поля модели; нет данных — ставь "" или [].
4. \*\*Для metaMetaType используй только IRI из переданного списка metaMetaList (новые не добавляй).\*\*
5. Проверь:
• нет дублирования значений;
• definition\_ru начинается с BORO-класса;
• уровни Bloom / SOLO / DOK выбраны по минимальной компетенции.
6. Выведи один JSON строго по заданной схеме, без внешних знаний и комментариев.
7. Не добавляй ничего от себя: никаких внешних знаний, предположений, пересказов других глав, ссылок на интернет и т.д.
### 2. Формат входа/выхода
\* Input — тексты руководств (русский).
\* Output — единственный JSON-объект (UTF-8, без комментариев).
