# Knowledge Index / Индекс знаний

Персональный индекс знаний — репозиторий для структурированного хранения, организации и повторного использования накопленных знаний.

## Назначение

Этот репозиторий решает проблему разрозненности личных знаний, превращая заметки, конспекты, статьи и идеи в единый структурированный индекс с возможностью семантического поиска.

## Концепция

Система основана на [Концепции Индексатора знаний](https://github.com/aisystant/ecosystem-development/blob/main/0.OPS/0.9.Inbox/%D0%9A%D0%BE%D0%BD%D1%86%D0%B5%D0%BF%D1%86%D0%B8%D1%8F%20%D0%98%D0%BD%D0%B4%D0%B5%D0%BA%D1%81%D0%B0%D1%82%D0%BE%D1%80%D0%B0%20%D0%B7%D0%BD%D0%B0%D0%BD%D0%B8%D0%B9.md) и адаптирована для персонального использования.

### Принципы

- **Версионирование** — все знания хранятся в Git, что обеспечивает историю изменений
- **Нормализация** — контент разбивается на чанки (единицы знаний) с метаданными
- **Структурированность** — единый формат для всех типов контента
- **Переиспользуемость** — возможность находить и связывать знания между собой

## Структура репозитория

```
/
├── README.md
├── docs/              # Персональные заметки и эссе (по годам)
├── posts/             # Публичные посты и заготовки для клуба
├── sources/           # Первичные источники
├── data/              # Данные
└── scripts/           # Скрипты обработки
```

### Публичные посты (`posts/`)

Все публичные посты (и заготовки) для клуба и других площадок хранятся в `posts/`.

**Конвенция имён:** `YYYY-MM-DD-english-name.md`

**Обязательный frontmatter:**
```yaml
---
type: post
status: draft | published
created: YYYY-MM-DD
target: club | blog | other
source_knowledge: <ссылка на Pack/Framework, откуда взято знание>
---
```

Посты — downstream-продукт. Знание берётся из Pack'ов и Framework'ов, пост — его адаптация для аудитории.

## Формат чанка

Каждая единица знаний оформляется как markdown-файл с YAML-метаданными:

```markdown
---
title: Название концепции
source: Источник (книга, статья, курс)
tags: [тег1, тег2, тег3]
created: 2026-01-18
updated: 2026-01-18
status: draft | active | archived
---

# Название

Содержание единицы знаний (до 10 000 символов).

## Связанные концепции

- [[Ссылка на другую концепцию]]
```

## Рабочий процесс

1. **Сбор** — новые материалы попадают в `0.Inbox/`
2. **Обработка** — материал изучается, выделяются ключевые идеи
3. **Нормализация** — создаются чанки с метаданными
4. **Размещение** — чанки распределяются по соответствующим разделам
5. **Связывание** — добавляются ссылки на связанные концепции

## Поиск и использование

Индекс поддерживает несколько способов поиска:

- **Навигация по структуре** — просмотр папок и файлов
- **Полнотекстовый поиск** — поиск по содержимому (через GitHub или локально)
- **Поиск по тегам** — фильтрация по меткам в метаданных
- **Семантический поиск** — поиск по смыслу (при наличии векторного индекса)

## Развитие

Планируемые этапы развития индекса:

1. **MVP** — ручное добавление и организация контента
2. **Автоматизация** — скрипты для импорта из различных источников
3. **Индексация** — создание векторных представлений для семантического поиска
4. **Интеграция** — подключение к AI-ассистентам для работы с базой знаний
